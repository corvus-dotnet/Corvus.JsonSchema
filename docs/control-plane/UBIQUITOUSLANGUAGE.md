# Ubiquitous language

A living glossary of the terms this project uses, in the Domain-Driven Design sense: one name per
concept, used consistently in code, documents, APIs, UI copy, and conversation. **Maintained at the
root of the Arazzo design-docs folder; update it in the same change that introduces or refines a
term.** If a term here conflicts with usage elsewhere, this document wins and the other usage should
be fixed. Known conflicts and their resolutions are recorded in
[Normalization decisions](#normalization-decisions) at the end.

Sections, and the entries within each section, are ordered alphabetically. Where a definition uses
another entry's term (in exact or inflected form), its first use links to that entry.

Terms marked **(proposed)** are being introduced by in-flight design work and settle when that
design lands.

## API mechanics

| Term | Definition |
|------|------------|
| <a id="keyset-pagination"></a>**Keyset pagination** | The project-wide list idiom: `limit` + opaque `pageToken` in, `nextPageToken` out, ordered by a stable per-store key. Every list endpoint *and its store* is paged from the start. |
| <a id="problem-details"></a>**Problem details** | Errors are RFC 9457 `application/problem+json` (`type`, `title`, `status`, `detail`, `instance`); clients branch on status, never parse strings. |

## Arazzo document model

| Term | Definition |
|------|------------|
| <a id="action"></a>**Action** | A success action (`end`, `goto`) or failure action (`end`, `goto`, `retry` with `retryAfter`/`retryLimit`), attached to a [step](#step) or inherited from the [workflow](#workflow) level. |
| <a id="arazzo-document"></a>**Arazzo document** | A document conforming to the Arazzo specification (1.0/1.1): `info`, `sourceDescriptions`, one or more `workflows`, optional `components`. The unit the [workflow designer](#workflow-designer) edits and the [catalog](#catalog) stores. |
| <a id="components"></a>**Components** | The [Arazzo document](#arazzo-document)'s reusable library (`inputs`, `parameters`, `successActions`, `failureActions`) referenced via reusable objects. |
| <a id="criterion"></a>**Criterion** | A success/assertion condition (`simple`, `regex`, `jsonpath`, `xpath`) with optional `context` [runtime expression](#runtime-expression). Used in [step](#step) `successCriteria`, in [action](#action) `criteria`, and in [scenario](#scenario) `$outputs` expectations (judged by the same compiler). |
| <a id="implicit-dependency"></a>**Implicit dependency** (step) | An ordering edge a [step](#step) imposes not by declaring `dependsOn` but by reading another same-[workflow](#workflow) step's data through a `$steps.<id>.*` [runtime expression](#runtime-expression) (Arazzo 1.1 §5.8.5.2.4): the referenced step MUST complete first. The generator folds implicit dependencies into the same topological order as explicit `dependsOn` (`WorkflowExecutorEmitter.TopologicallyOrder`); a cycle in the combined graph is an error. Contrast **explicit dependency** (a `dependsOn` entry). |
| <a id="operation-surface"></a>**Operation surface** | The set of operations (OpenAPI) or channel operations (AsyncAPI) a [source document](#source-document) exposes; what a user browses to create [steps](#step). Described by `OperationDescriptor` / `AsyncApiChannelDescriptor`. |
| <a id="runtime-expression"></a>**Runtime expression** | An Arazzo `$…` expression (`$inputs`, `$steps`, `$outputs`, `$response`, `$statusCode`, …) evaluated during execution; parsed by `ArazzoExpression`. |
| <a id="source-description"></a>**Source description** | An entry in the [Arazzo document](#arazzo-document)'s `sourceDescriptions`: a named reference (`name`, `url`, `type` = `openapi` \| `asyncapi` \| `arazzo`) to a [source document](#source-document). In-document; distinct from a *source document* (the file itself) and a *[registered source](#registered-source)* (a `/sources` registry entry). |
| <a id="source-document"></a>**Source document** | The actual OpenAPI/AsyncAPI/Arazzo file a [workflow](#workflow)'s [steps](#step) bind against; stored inside the [package](#package) as `sources/<name>.json`. |
| <a id="step"></a>**Step** | One unit of work in a [workflow](#workflow). Binds to exactly one of: an OpenAPI operation (`operationId` or `operationPath`), another workflow (`workflowId`), or an AsyncAPI channel (`channelPath`, 1.1). Carries `parameters`, `requestBody`, `successCriteria`, `onSuccess`/`onFailure` [actions](#action), and `outputs`. |
| <a id="workflow"></a>**Workflow** | A named sequence of [steps](#step) within an [Arazzo document](#arazzo-document) (`workflowId`, `inputs` schema, `steps`, workflow-level `successActions`/`failureActions`, `outputs`, `parameters`, `dependsOn`). |

## Catalog and lifecycle

| Term | Definition |
|------|------------|
| <a id="baked-schemas"></a>**Baked schemas** | The precomputed `metadata/schemas.json` [package](#package) entry: per-[step](#step) schema metadata (typed inputs, resolved output types) that drives `/validate` and the UI's typed forms without re-parsing [source documents](#source-document). |
| <a id="base-workflow-id"></a>**Base workflow id** | The unversioned [workflow](#workflow) identity (`nightly-reconcile`) under which [versions](#version) group. |
| <a id="catalog"></a>**Catalog** | The control plane's immutable, [content-hashed](#content-hash), versioned store of [workflow](#workflow) [packages](#package), with governance metadata and search. |
| <a id="content-hash"></a>**Content hash** | SHA-256 over the RFC 8785 canonical form of the logical `{workflow, sources}` content — framing-independent, stable across repacks. The [version](#version)'s identity and the [executor](#executor)'s integrity binding. |
| <a id="draft"></a>**Draft** | **A [version](#version) not yet made [available](#availability) in any [environment](#environment).** A derived state (its availability set is empty), not a stored flag. A draft is exercised by [scenarios](#scenario) in the [workflow designer](#workflow-designer), then [promoted](#promotion) through environments. Once available anywhere it is no longer a draft. |
| <a id="executor-manifest"></a>**Executor manifest** | The `metadata/executor-manifest.json` [package](#package) entry: target framework, assembly digest, package-hash binding, entry type. Verified before an [executor](#executor) loads. |
| <a id="obsolete"></a>**Obsolete** | The retired governance status of a [version](#version) (`Active` → `Obsolete`); still addressable until [purged](#purge), never deletable while [runs](#run) reference it. |
| <a id="package"></a>**Package** (`.awp`) | The self-contained, deterministic, [content-hashed](#content-hash) container for a [version](#version): the [Arazzo document](#arazzo-document), every referenced [source document](#source-document), and named metadata entries. A complete input to code generation. |
| <a id="publish"></a>**Publish** | The act of minting a catalog [version](#version) from a [working copy](#working-copy), together with its [scenarios](#scenario) and the [evidence](#evidence) of their successful execution. A published-but-unpromoted version is a [draft](#draft). (Serving a version at an endpoint is *hosting*, not publishing — see normalization E.) |
| <a id="purge"></a>**Purge** | The destructive bulk-reap of [obsolete](#obsolete) [versions](#version), or of old completed/cancelled [runs](#run), that nothing references. Gated by a `:purge` [capability scope](#capability-scope) and bounded by purge [reach](#reach). |
| <a id="runnable"></a>**Runnable** | A [version](#version) that carries a compiled [executor](#executor) (code generation and compile succeeded at add). Only runnable versions can be [triggered](#trigger). |
| <a id="version"></a>**Version** (catalog version) | One immutable release of a [base workflow](#base-workflow-id): the [package](#package) plus governance metadata. The store assigns `versionNumber` and rewrites the workflow id to `{base}-v{n}` (the **workflow-id rewrite**). Content never changes after add; only governance metadata is mutable. |
| <a id="working-copy"></a>**Working copy** | The mutable [Arazzo document](#arazzo-document) under edit in the [workflow designer](#workflow-designer), with its [scenarios](#scenario). Durably saveable on its own, as many times as needed, without ever minting a catalog [version](#version) — so development iteration does not pollute the [catalog](#catalog). Persisted in the designer workspace (and optionally a Git branch); *[publish](#publish)* mints a catalog version from it. Prefer this term over "draft save": **[draft](#draft)** is reserved for an unpromoted catalog version. |

## Design, test, and validation

| Term | Definition |
|------|------------|
| <a id="advanced-node"></a>**Advanced node** | A schema construct the [schema editor](#schema-editor)'s visual tier summarises rather than fully forms — a keyword it does not render, `not`, `patternProperties`, a `$ref` carrying siblings, and the like. Shown as a labelled row that preserves the raw subschema and stays JSON-editable, so a round-trip never destroys or reorders it. |
| <a id="change-classification"></a>**Change classification** | The label a [comparison](#comparison) assigns each element: *added* (present only on the right), *removed* (present only on the left), *changed* (present on both, content differs), *renamed* (a [step](#step) matched by binding under a new id — see [rename](#rename-step)), *moved* (a step reordered, content identical — never *changed*). |
| <a id="comparison"></a>**Comparison** (workflow) | A structured diff of two [workflow](#workflow) [versions](#version) (`diffWorkflowPair`): [steps](#step), flow edges, and workflow-level surfaces classified as added / removed / changed / renamed / moved (the [change classification](#change-classification)) on top of the graph projection and the document model's identity-aware structural diff. Rendered by `<arazzo-workflow-compare>` as side-by-side, overlay, or text (the [comparison modes](#comparison-mode)). |
| <a id="comparison-mode"></a>**Comparison mode** | The [comparison](#comparison)'s presentation, per dialog instance: **Side by side** (two surfaces), **Overlay** (one union canvas, the base [version](#version) solid and the other side's exclusives translucent as *ghosts*), or **Text** (a CodeMirror MergeView over the two serialized documents). |
| <a id="debug-run"></a>**Debug run** | The durable, forward-only server-side entity a [debug session](#debug-session) drives (REST `debugRuns` tag, §18): started from a [working copy](#working-copy), run/paused/stepped, resumable, and cancellable/deletable. Backed by a real [executor](#executor) whose transport is either a [scenario](#scenario)'s [mock transport](#mock-transport) or a live transport in a development-class [environment](#environment); forward-only means time-travel is a replay over the recorded [trace](#trace), not a rewind of the run. |
| <a id="debug-session"></a>**Debug session** | An interactive [simulation](#simulation) in the [workflow designer](#workflow-designer): run/pause/step a [workflow](#workflow) on the design surface against a [scenario](#scenario)'s [mock transport](#mock-transport) and [virtual clock](#virtual-clock), with breakpoints on [steps](#step), live state inspection, expression evaluation against the paused context, [trigger injection](#trigger-injection), and time-travel over the recorded [trace](#trace). |
| <a id="evidence"></a>**Evidence** | The durable record that a [version](#version)'s [scenarios](#scenario) executed successfully: the scenario set (by [content hash](#content-hash)), per-scenario outcomes, and the engine/version identity, attached at [publish](#publish) and surfaced by the [catalog](#catalog). |
| <a id="guarded-json-editor"></a>**Guarded JSON editor** | A parse-gated raw-JSON textarea whose edits apply only when the buffer parses, holding the last valid value otherwise (`wireGuardedJson`). Per host, a blank buffer either deletes the value or holds the last valid one. The single shared idiom behind the [schema editor](#schema-editor)'s JSON tier, the payload editor, and the document inspector. |
| <a id="merge-target"></a>**Merge target** | The side of a [comparison](#comparison), opened with `mergeTarget: true`, whose document the host is editing (the [working copy](#working-copy)). It turns the comparison into an interactive merge and sources its document from the live model, never a snapshot. |
| <a id="mock-transport"></a>**Mock transport** | The simulated transport (`MockApiTransport`) that answers the operations a [workflow](#workflow)'s [source documents](#source-document) expose during a [simulation](#simulation), with scripted responses instead of real endpoints. (Not a *[runner](#runner)* — the [executor](#executor) is real; its transport and clock are simulated.) |
| <a id="rename-step"></a>**Rename** (step) | A [step](#step) matched across [versions](#version) by binding equality (same `operationId` / `operationPath` / `channelPath`+action / `workflowId`) under a different `stepId`. Presented as one changed step with a rename note, never a remove plus an add. |
| <a id="scenario"></a>**Scenario** | A named, declarative test for a [workflow](#workflow): inputs, scripted [mock-transport](#mock-transport) behaviour, triggers, [virtual-clock](#virtual-clock) advances, and expected outcomes/paths/outputs. Associated with the [version](#version) under design; carried forward when a new version of a published workflow is edited. |
| <a id="scenario-runner"></a>**Scenario runner** | The CLI (`scenarios run`) that executes a globbable set of [scenarios](#scenario) against a [workflow](#workflow) — in-process via the [simulator](#simulation) (standalone, no control plane) or against a control plane — with CI-grade reports and exit codes; wrappable as a GitHub Action. |
| <a id="schema-editor"></a>**Schema editor** | The typed JSON-Schema authoring form (`<arazzo-schema-editor>`) for a [workflow](#workflow)'s `inputs` and the [components](#components) library's input schemas, in place of a raw textarea. Edits a *subset* of JSON Schema as typed property rows and first-class `oneOf`/`anyOf`/`allOf` combiner nodes, stays lossless over the rest via **[advanced nodes](#advanced-node)**, offers a Form \| JSON toggle over the **[guarded JSON editor](#guarded-json-editor)**, a live typed preview, and a shared-type reference picker. |
| <a id="simulation"></a>**Simulation** | Executing a [workflow](#workflow)'s compiled [executor](#executor) against the [mock transport](#mock-transport) and [virtual clock](#virtual-clock), producing a [trace](#trace). Deterministic and replayable. |
| <a id="take-keep"></a>**Take / Keep** | The interactive-merge verbs on a change-list entry when there is a [merge target](#merge-target). *Take* adopts the other side's state for that one entry (the component emits `change-accepted`/`merge-text-applied`; the host applies it to its model, undoably). *Keep* marks the entry reviewed for the session — the [working copy](#working-copy) IS the merge state, so a change not taken is kept by definition; nothing is persisted. |
| <a id="trace"></a>**Trace** | The structured record of one [simulation](#simulation): the path taken, per-[step](#step) requests/responses, [criterion](#criterion) results, retries, [waits](#wait), outputs, and final outcome. |
| <a id="trigger-injection"></a>**Trigger injection** | Delivering a message trigger into a [debug run](#debug-run) that is `Suspended` on a message [wait](#wait) (REST `injectDebugRunMessage`): the payload is handed straight to the suspended run's correlated wait to [resume](#resume) it — nothing is published to a broker — so a message-driven [workflow](#workflow) can be driven interactively without an out-of-band producer. |
| <a id="virtual-clock"></a>**Virtual clock** | The simulated `TimeProvider` a [simulation](#simulation) runs against, so timer [waits](#wait) and retries elapse deterministically without real time passing. |
| <a id="workflow-designer"></a>**Workflow designer** | The UI for authoring an [Arazzo document](#arazzo-document): a visual design surface (diagram area + context inspector) and a text editor over the same [working copy](#working-copy). The design surface doubles as the interactive debugger for [simulations](#simulation) (the [debug session](#debug-session)) — it is an instrument, not an illustration. |

## Environments and promotion

| Term | Definition |
|------|------------|
| <a id="availability"></a>**Availability** | The many-to-many relation "[version](#version) V is available in [environment](#environment) E". Additive; governed by the target environment's [administrators](#environment-administrator). |
| <a id="availability-matrix"></a>**Availability matrix** | The [version](#version) × [environment](#environment) grid showing each pair as [available](#availability), promotable, or not [ready](#readiness), with direct make/withdraw or request-[promotion](#promotion) actions. |
| <a id="availability-request"></a>**Availability request** | A request that a [version](#version) be made [available](#availability) in an [environment](#environment), routed to that environment's [administrators](#environment-administrator)' [approver inbox](#approver-inbox) (approve/deny/withdraw). The environment-scoped sibling of an [access request](#access-request). |
| <a id="environment"></a>**Environment** | A first-class governed deployment target ([administrator](#environment-administrator) set, audit) that [versions](#version) are made [available](#availability) in and [credentials](#credential) are bound per. |
| <a id="environment-administrator"></a>**Environment administrator** | A member of an [environment](#environment)'s governing set; approves [availability](#availability-request) and [runner-authorization](#runner-environment-binding) requests for that environment. The environment-scoped counterpart of a workflow [administrator](#administrator). |
| <a id="promotion"></a>**Promotion** | Making a [version](#version) [available](#availability) in an [environment](#environment), either directly ([environment administrator](#environment-administrator)) or via an [availability request](#availability-request) through the [approver inbox](#approver-inbox). [Readiness](#readiness)-gated. |
| <a id="readiness"></a>**Readiness** | The gate on [promotion](#promotion): every [registered source](#registered-source) the [version](#version) references must have a [usable](#usability) [credential](#credential) in the target [environment](#environment), and — where the environment sets `requireEvidence` — the version's [publish](#publish) [evidence](#evidence) must attest a **green suite** (at least one [scenario](#scenario) ran, none failed). Absent evidence, or an empty suite, counts as unevidenced and is refused there. Per-environment and default-off, so environments that never opt in keep the credential-only gate. |

## Execution

| Term | Definition |
|------|------------|
| <a id="checkpoint"></a>**Checkpoint** | The single small JSON document per [run](#run) that *is* the entire resumable state: [cursor](#cursor), [step](#step) outputs, retry counters, correlation tokens, inputs, [wait](#wait), [fault](#fault), history, etag. |
| <a id="correlation-id"></a>**Correlation id** | The key a message [wait](#wait) matches on, so an inbound message [resumes](#resume) exactly the [run](#run) that awaits it. |
| <a id="cursor"></a>**Cursor** | The scalar state-machine position in a [run](#run)'s [checkpoint](#checkpoint) identifying which [step](#step) runs next; the core of [resume](#resume). |
| <a id="execution-backend"></a>**Execution backend** | The pluggable isolation model a [runner](#runner) dispatches a [run](#run) to: in-process collectible load context, per-run micro-guest, serverless function, or container — an isolation ↔ latency ↔ cost choice. |
| <a id="executor"></a>**Executor** | The compiled assembly generated from a [package](#package) that runs the [workflow](#workflow); loaded per [version](#version) with digest verification against its [executor manifest](#executor-manifest). |
| <a id="fault"></a>**Fault** | A [run](#run) is *faulted* when a [step](#step) errors and `failureActions` do not resolve it: terminal-but-recoverable. The fault record captures step id, attempt, error, time, and retriability. |
| <a id="lease"></a>**Lease** | The advisory single-owner, TTL-bounded claim that stops a horizontally-scaled [runner](#runner) fleet double-advancing one [run](#run). |
| <a id="nested-trace-record"></a>**Nested trace record** (`subTrace`) | The complete [trace](#trace) of one [sub-workflow](#sub-workflow-step) execution, attached to its parent [step](#step)'s record: `{workflowId, outcome, steps, stepsExecuted, …}` — the same trace shape, recursively. The root trace carries no `workflowId`; sub-traces always do. |
| <a id="resume"></a>**Resume** | Continuing a [suspended](#wait)/[faulted](#fault) [run](#run) by mode: retry faulted [step](#step), rewind, skip, or **state patch** (an RFC 6902 patch to the run context, then retry). |
| <a id="run"></a>**Run** | One execution of a [workflow](#workflow) [version](#version) (`Pending` → `Running` → `Suspended`/`Completed`/`Cancelled`/`Faulted`), durable and [resumable](#resume). |
| <a id="runner"></a>**Runner** | An execution host that claims and executes [runs](#run); self-registers and heartbeats to the [runner registry](#runner-registry). |
| <a id="runner-environment-binding"></a>**Runner ↔ environment binding** | One [environment](#environment) per [runner](#runner) (many runners per environment): a runner is provisioned for exactly one environment's [credentials](#credential)/network and must be authorized by that environment's [administrators](#environment-administrator) before it is dispatchable. |
| <a id="runner-registry"></a>**Runner registry** | The store-backed table of [runners](#runner) and the [versions](#version) they host; a runner missing heartbeats goes **Stale**, past TTL it is pruned and its [runs](#run) become claimable. |
| <a id="scoped-step-path"></a>**Scoped step path** | The address of a [step](#step) inside nested [sub-workflows](#sub-workflow-step): `parentStepId/childStepId[/...]` (slash-delimited; safe because Arazzo stepIds match `^[A-Za-z0-9_\-]+$`). A bare stepId addresses a root step only. Used by breakpoints (`pause.beforeSteps`), run-to-here (`until.beforeStepId`), and the [trace](#trace)'s `pausedBefore`. |
| <a id="step-into-step-over"></a>**Step-into / step-over** | The debugger verbs over a [sub-workflow step](#sub-workflow-step): step-over treats it as one unit (the parent record's status/outputs); step-into descends into its [nested trace record](#nested-trace-record), the canvas following, breadcrumb back. Descent over a recorded prefix is client-side [trace](#trace) navigation; acquiring a deeper stop is a stateless replay with a scoped `until`. [Resume](#resume) verbs and the durable [cursor](#cursor) stay parent-scoped. |
| <a id="sub-workflow-step"></a>**Sub-workflow step** | A [step](#step) bound to another [workflow](#workflow) (`workflowId` binding). Executes inline within the parent's [run](#run): no child run entity, no separate [cursor](#cursor), its result materialized on the parent step's record. |
| <a id="trigger"></a>**Trigger** | How a fresh [run](#run) is started (as opposed to [resuming](#resume) a suspended one): the HTTP, message, or schedule trigger surface. |
| <a id="wait"></a>**Wait** | Why a [run](#run) is `Suspended`: a timer (`dueAt`) or a correlated message (`channel` + [`correlationId`](#correlation-id)). |

## Security and governance

Every access decision has **two planes**: WHO can do WHAT (capability scopes) and WHERE (reach).
Both must pass.

| Term | Definition |
|------|------------|
| <a id="access-request"></a>**Access request** | A control-plane resource any principal submits to ask for elevated access to a specific [workflow](#workflow); routed to that workflow's [administrators](#administrator); approval writes an [entitlement](#entitlement). A request can never target a third party. |
| <a id="administrator"></a>**Administrator** | A member of a [workflow](#workflow)'s or [environment](#environment)'s governing set (see [environment administrator](#environment-administrator)); the only third-party grant in the access model. |
| <a id="approver-inbox"></a>**Approver inbox** | The queue showing an approver every pending [access](#access-request)/[availability](#availability-request)/[runner](#runner-environment-binding) request across everything they administer, without naming a subject first. |
| <a id="capability-scope"></a>**Capability scope** | A token capability tier (`catalog:read`, `runs:write`, `credentials:write`, …) gating API operations. Say "capability scope", not bare "scope", where ambiguity is possible. |
| <a id="credential"></a>**Credential** | A per-[registered-source](#registered-source), per-[environment](#environment) binding of a **[secretRef](#secretref)** plus non-secret configuration. Its **status** (`Valid`/`ExpiringSoon`/`Expired`) is derived on read; rotation is re-pointing the reference. |
| <a id="digest"></a>**Digest** (identity digest) | The canonical digest over a stamped identity set — equal iff set-equal. The stable key [administrators](#administrator) are indexed by and removed by; [membership](#membership) lookups enumerate a caller's subset digests against it. |
| <a id="eligibility"></a>**Eligibility** | The PIM-style "you may self-elevate this" grant (*approve-as-eligible* on an [access request](#access-request)): durable permission to activate access JIT without re-approval. Eligible is not active — wielding requires an explicit, audited, time-boxed activation. |
| <a id="entitlement"></a>**Entitlement** | A per-principal stored grant (never in the IdP) binding a subject claim to a [capability scope](#capability-scope) and a [reach](#reach), optionally time-boxed. Capability = token claims ∪ stored entitlements. |
| <a id="external-schema-reference"></a>**External schema reference** | An inputs-schema `$ref` to an attached `jsonschema` document (never a `sourceDescription` — the Arazzo spec pins that enum). The PREFERRED base is the document's absolute root `$id` (its canonical identity); the virtual relative path `schemas/<name>#<pointer>` is the fallback for an `$id`-less document. Packaged alongside the [source documents](#source-document) at [publish](#publish) and registered with the generation resolver under both identities. A reference matching no attached document is a validation error. |
| <a id="grant-binding"></a>**Grant binding** | A `{claimType, claimValue}` → per-verb [reach](#reach) mapping (each verb `Denied`, `Unrestricted`, or a set of [rule](#security-rule) names), optionally pinning further [identity dimensions](#identity-dimension) (`additionalClauses`, matched as a conjunction by [membership](#membership)). The API/wire name is *security binding*; docs and UI say **grant binding**. |
| <a id="grantee"></a>**Grantee** | A real person/team/role/workflow named by governance surfaces, resolved (via the [merged grantee search](#merged-grantee-search), [principal-directory](#principal-directory) search, observed-identity typeahead, or a validated well-known subject id) to the exact **[resolved identity](#resolved-identity)** the deployment stamps. |
| <a id="identity-dimension"></a>**Identity dimension** | One `{dimension, value}` key of a stamped [`sys:`](#internal-sys-tag) identity (e.g. `sys:tenant`, `sys:sub`, `sys:iss`). An **ambient identity dimension** is derived from request context (host, route, gateway header) rather than the token, stamped identically at runtime and grant-authoring moments. |
| <a id="internal-sys-tag"></a>**Internal (`sys:`) tag** | A reserved-prefix [reach label](#reach-label) stamped by the deployment shell: immutable, client-invisible, rejected on user input; the unforgeable key administration and [grants](#grant-binding) rely on. |
| <a id="membership"></a>**Membership** (identity matching) | The governing match rule for administration, [grant-binding](#grant-binding) applicability, and credential [usability](#usability): a principal matches a named identity iff the principal's whole stamped [`sys:`](#internal-sys-tag) identity **contains** it (subset test) — never by exact set-equality, and never against raw token claims. |
| <a id="merged-grantee-search"></a>**Merged grantee search** | The [grantee](#grantee)-search default when a [principal directory](#principal-directory) is configured: the directory (every searchable kind) unioned with the observed-identity typeahead, directory-preferred on a `(kind, value)` collision, as one bounded unpaged result set. A directory failure degrades a merged search to observed results; only an explicit `source=directory` search surfaces it (502). |
| <a id="partial-identity"></a>**Partial identity** | A [grantee](#grantee) resolution that yields only some of the [identity dimensions](#identity-dimension) the deployment stamps. Flagged because under [membership](#membership) matching a partial identity is *broader* than intended — every principal whose stamped identity contains those dimensions matches it — and authoring a grant on one raises the broadening-overlap advisory naming any subsumed [grantees](#grantee). |
| <a id="persona"></a>**Persona** | A named operator archetype the UX targets (operator/SRE, publisher/owner, connections admin, security admin, approver, requester); the demo's persona selector drives [capability-scope](#capability-scope) gating from one source of truth. |
| <a id="principal-directory"></a>**Principal directory** | The pluggable directory seam (LDAP/AD, Keycloak, SCIM, Entra ID, Okta, Google) that searches people/teams/roles and returns [resolved identities](#resolved-identity), each expanded to full [membership](#membership) (a `sys:group`/`sys:role` per membership). It resolves *who*, never *what they may do*. |
| <a id="reach"></a>**Reach** | Row-level visibility: which records a caller can see/affect per verb (read/write/purge), defined by [security rules](#security-rule) and [grant bindings](#grant-binding). |
| <a id="reach-label"></a>**Reach label** | The umbrella term for the `{key, value}` labels [reach rules](#security-rule) match against: **security tags** on catalog [versions](#version) and [runs](#run) (inherited by runs from their version), **management tags** on [environments](#environment), [registered sources](#registered-source), and [credentials](#credential). Never say bare "tags" — *user tags* are the separate free-form display/filter metadata. |
| <a id="registered-source"></a>**Registered source** | An entry in the control plane's `/sources` registry: a named [source document](#source-document) registered once and referenced by [workflows](#workflow), with per-[environment](#environment) [credentials](#credential). A `jsonschema` source registers a JSON Schema document for **[external schema references](#external-schema-reference)**. |
| <a id="resolved-capability-view"></a>**Resolved capability view** | The Access Overview's `capabilities` section: the [capability scopes](#capability-scope) a [grantee](#grantee)'s matched [bindings](#grant-binding) confer, resolved exactly as the runtime resolver does — expired bindings confer nothing; an [eligible](#eligibility)-only binding shows as *eligible*, not active. |
| <a id="resolved-identity"></a>**Resolved identity** | The full stamped [`sys:`](#internal-sys-tag) identity a [grantee](#grantee) resolves to: the principal's own [identity dimensions](#identity-dimension) plus a `sys:group`/`sys:role` per membership (full membership expansion), exactly as the deployment's runtime resolver stamps it. Grants are authored against it and [membership](#membership) matching compares against it, so the authoring and runtime moments cannot drift. |
| <a id="secretref"></a>**secretRef** | A scheme'd *reference* to secret material in an external store (`keyvault://`, `vault://`, `env://`, …) — never the material itself. Only [runners](#runner) resolve it, at bind time. |
| <a id="security-rule"></a>**Security rule** | A named, reusable boolean row-filter expression over [reach labels](#reach-label), written in the `simple`-[criterion](#criterion) grammar (e.g. `tenant == $claim.tenant`). The reusable WHERE vocabulary a [grant binding](#grant-binding) points at. |
| <a id="tenant-shell"></a>**Tenant shell** | A deployment wrapper that ANDs a mandated [rule](#security-rule) (e.g. `sys:tenant == $claim.tenant`) and [internal-tag](#internal-sys-tag) stamping into every decision, so multi-tenant isolation is inescapable and users can only narrow within it. |
| <a id="usability"></a>**Usability** | The label-superset test (`IsUsableBy`) governing which [runs](#run) may use a [credential](#credential): the run must carry all the binding's usage tags. Distinct from [reach](#reach) (superset test, not rule match). |

## Normalization decisions

Conflicts found by the 2026-07 docs sweep, and the canonical resolutions. Renames were marked ⚠
while pending; the 2026-07 normalization pass ([`terminology-normalization.md`](./terminology-normalization.md))
has since applied them all, so no ⚠ remain.

| # | Conflict | Resolution |
|---|----------|------------|
| A | "Scope" means (1) capability tier, (2) the per-verb grant value `Scoped`, (3) the UI's "Scopes" panel which actually edits security *rules* (`ux-review.md` §"Scopes = security rules"). | (1) is canonical: **capability scope**. Say **scoped reach** for (2). The UI surface is **Rules**: primary tag `arazzo-rules-panel` with `arazzo-scopes-panel` kept as a deprecated alias; `ui-design.md`, `security-ui-design.md`, `ux-review.md` copy updated. |
| B | The grant object has four names: grant / security binding / grant binding / claim→rule binding. | **Grant binding** is canonical in docs/UI; **security binding** is recorded as the API/wire synonym. |
| C | "Source" is overloaded: in-document description, the file, the registry entry, and the UI "Sources" tab that actually manages credentials. | Three precise terms — **source description**, **source document**, **registered source**. The credential-management tab is now **Connections** (was "Sources"); demo, `ui-design.md`, `ux-review.md` updated. |
| D | "Draft" is also allocation-campaign jargon for a pooled document being built at a store's write leaf (`allocation-matrix.md`, `ValidateDraft`). | **Draft** is the lifecycle term only. Scope-qualify the allocation jargon as **write draft** (or rename to *builder document*) in campaign docs/code as touched. |
| E | "Publish" as mint-a-version vs the `catalog-design.md` "publish/hosting service" that serves a version at an endpoint. | **Publish** = mint a catalog version. The endpoint-serving service is the **hosting service** (`catalog-design.md` future-phase section updated). |
| F | Bare "tags" ambiguity (user tags vs security tags vs management tags). | Always qualify: **user tags** / **security tags** / **management tags**; **reach labels** is the umbrella. |
| G | `ux-review.md` labels the Access area "Reach scopes", re-fusing the two planes. | Say **reach rules**; avoid "reach scopes". |
| H | "Readiness" described as a soft check in one place and a hard gate in another (`ux-review.md`). | Readiness is a **hard gate** on promotion and the add-workflow wizard; views may *display* readiness, but promotion never bypasses it. |
