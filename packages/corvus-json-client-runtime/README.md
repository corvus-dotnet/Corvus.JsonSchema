# @endjin/corvus-json-client-runtime

The byte-native HTTP transport runtime for TypeScript OpenAPI clients generated by the
**Corvus.Text.Json** code generator (`corvusjson openapi-client --engine TypeScript`).

A generated client is pure composition/decomposition code: it serializes parameters and
request bodies to bytes, sends them through a transport, and decomposes the response bytes
back into typed models. This package is that transport seam — the wire, the middleware, and
the authentication — kept out of the generated code so the same client runs unchanged in a
browser, on the edge, or on Node.

It is independent of [`@endjin/corvus-json-runtime`](../corvus-json-runtime): a generated
client imports the **model** runtime for typed request/response bodies and **this** package
for the wire.

## What's in it

The exports a **consumer** (or generated client) reaches for:

| Export | Purpose |
| --- | --- |
| `FetchApiTransport` | default transport over the platform `fetch` (browsers, edge, Node 18+) — `new FetchApiTransport({ baseUrl, authenticationProvider?, handlers? })` |
| `NodeApiTransport` | Node transport (`undici`, with a `node:http` fallback) — exported from `@endjin/corvus-json-client-runtime/node` |
| `AbstractApiTransport` | base class that builds the wire request synchronously and runs the middleware pipeline; extend it for a custom transport |
| `bearerToken`, `apiKey`, `basicAuth` | `AuthenticationProvider` factories (bearer token or token factory; API key in header/query/cookie; HTTP basic) |
| `retryHandler`, `redirectHandler`, `timeoutHandler`, `defaultHandlers`, `compose` | composable middleware over `WireRequest → Promise<WireResponse>` (Retry-After + backoff, follow/strip-auth redirects, per-attempt `AbortSignal`) |
| `ValidationMode`, `OperationMethod` | the response-validation levels and HTTP method enum the generated client references |

The serializers (percent-encoding, parameter `style` writers, form-urlencoded, multipart,
NDJSON/SSE readers, JSON-pointer) and the `ApiRequest`/`ApiResponse`/`ApiTransport`/`WireRequest`/
`WireResponse` contracts are also exported; generated code imports them directly. See
[docs/typescript/openapi.md](../../docs/typescript/openapi.md) for the full client guide.

## Install & use

```sh
npm install @endjin/corvus-json-client-runtime
```

```ts
import { FetchApiTransport, bearerToken } from "@endjin/corvus-json-client-runtime";
import { ApiPetsClient } from "./generated/ApiPetsClient.js"; // your generated client

const transport = new FetchApiTransport({
  baseUrl: "https://petstore.example.com/v1",
  authenticationProvider: bearerToken(process.env.TOKEN!),
});

const client = new ApiPetsClient(transport);
const response = await client.listPets({ limit: 10 });
const names = response.match({ ok: (pets) => pets.map((p) => p.name), otherwise: () => [] });
```

### Node entry point

`@endjin/corvus-json-client-runtime/node` re-exports everything from the main entry **plus**
`NodeApiTransport` (undici-based). The main entry stays free of Node-only imports so browser
and edge bundles never pull in `undici`/`node:http`:

```ts
import { NodeApiTransport } from "@endjin/corvus-json-client-runtime/node";
```

ESM-only, `"type": "module"`, `"sideEffects": false` — tree-shakeable, and set `"type": "module"`
(or use `.mjs`) in consuming projects.

## Scripts

| Command | Purpose |
| --- | --- |
| `npm run build` | `tsc` → `dist/` (the `.` and `./node` entry points + `.d.ts` declarations) |
| `npm test` | `node --test` — unit-tests the transports (against a mock dispatch), serializers, middleware, and auth |

## Tooling convention

This is a non-shipping npm workspace node; it carries no Endjin StyleCop / Roslynator
treatment (those apply to the C# projects). The runtime is authored in `src/`.
