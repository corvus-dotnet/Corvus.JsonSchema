Example Recipes

Runnable examples demonstrating JSON Schema patterns with the V5 code generator.

001

Simple Data Objects

This recipe demonstrates how to define a simple data object of primitive values in JSON Schema, and how the Corvus.Text.Json TypeScript generator produces an idiomatic readonly interface plus a small, allocation-lean API to evaluate, read, build and mutate it.

 002

Validation

This recipe adds constraints to a data object (length, range, pattern, numeric step, format) and shows that the generated Registration.evaluate enforces them while the interface stays the plain shape.

 003

References

This recipe shows how a schema reuses a common type with $ref and $defs, and how that becomes one shared interface with its own generated API.

 004

Open and Closed Objects

This recipe shows the difference between an open object (unknown properties allowed) and a closed one (unevaluatedProperties: false), and how each is generated.

 005

Extending a Base Type

This recipe composes a base type with extra properties using allOf, the JSON Schema equivalent of inheritance.

 006

Constraining a Base Type

This recipe uses allOf not to add properties but to tighten a base type's constraints.

 007

Strongly Typed Arrays

This recipe shows a homogeneous array of objects: items types every element, and the array reads as a readonly T[].

 008

Arrays of Higher Rank

This recipe shows nested arrays (an array whose items are themselves arrays) generating a matrix type.

 009

Tuples

This recipe shows a fixed-length positional array (prefixItems with items: false) generating a TypeScript tuple type.

 010

Mix-in Types

This recipe composes several independent base types into one with a multi-member allOf.

 011

Unions

This recipe shows how oneOf becomes a TypeScript union with per-branch type guards and an exhaustive {Union}.match function, the idiomatic way to handle "one of several shapes".

 012

Discriminated Unions

This recipe shows a tagged union (a oneOf whose branches share a discriminant property) and the exhaustive dispatch it generates.

 013

String Enumerations

This recipe shows enum of strings generating a string-literal union, the idiomatic TypeScript enum.

 014

Numeric Enumerations

This recipe shows a numeric enum generating a numeric-literal union.

 015

Maps

This recipe shows an open map (additionalProperties with a value schema) generating an index-signature type.

 016

Mutation

This recipe covers the mutation surface in depth (build, patch and produce) over canonical UTF-8 JSON bytes.

 017

Conditional Schemas

This recipe shows if/then/else, applying constraints conditionally on a value's contents.

 018

Default Values

This recipe shows the default annotation and the two ways the generated TypeScript surfaces it.

 019

Format Validation

This recipe shows format keywords generating branded types with validating factories, and the accessors that convert a branded value into a strong runtime type.

 020

JSON Patch and Merge Patch

This recipe shows applying and building [RFC 6902 JSON Patch](https://datatracker.ietf.org/doc/html/rfc6902) and [RFC 7396 JSON Merge Patch](https://datatracker.ietf.org/doc/html/rfc7396) documents against a generated type's bytes.
