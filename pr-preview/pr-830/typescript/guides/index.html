Documentation

In-depth documentation for the TypeScript that Corvus.Text.Json generates from a JSON Schema: the read surface and validators, the byte-level mutation API, every JSON Schema construct mapped to its generated TypeScript, and the branded value types. New here? Start with the Getting Started walkthrough, or browse the worked examples.

Code generation

This is the reference for generating TypeScript from a JSON Schema: the corvusjson command, its options, the shape of the output, and the code-generation modes.

The type surface

How each JSON Schema construct maps to generated TypeScript.

Reading and evaluating

This guide covers the read side of a generated module: the readonly type surface and the boolean evaluators.

Value types — brands, numbers, dates & times

JSON's primitive types are coarse: every string is a string, every number a number.

Mutation

Parsed values are readonly, so you never change data in place.

JSON Patch and Merge Patch

This guide covers applying and building [RFC 6902 JSON Patch](https://datatracker.ietf.org/doc/html/rfc6902) and [RFC 7396 JSON Merge Patch](https://datatracker.ietf.org/doc/html/rfc7396) documents.

The runtime

The generated TypeScript imports a small runtime library for the parts the language cannot express on its own: format checks, exact numeric comparison, the byte-level edit engine, and Temporal conversions.
