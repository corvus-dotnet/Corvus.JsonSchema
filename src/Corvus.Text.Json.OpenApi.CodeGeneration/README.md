# Corvus.Text.Json.OpenApi.CodeGeneration

The shared, language-neutral core that generates API clients from OpenAPI (and AsyncAPI) specifications. It walks a specification once into a version-neutral operation model, then emits client code through a pluggable emitter. One core drives every supported spec version and both output languages, so 3.0, 3.1, and 3.2 are generated by a single symmetric pipeline rather than three parallel ones.

## Pipeline

```
OpenAPI/AsyncAPI spec
  -> parsed by the strongly-typed V5 model types (Corvus.Text.Json.OpenApi30/31/32)
  -> walked by OpenApiWalkerBase (a per-version OpenApiNNWalker subclass supplies the typed hooks)
  -> a version-neutral operation IR (OperationInfo: parameters, bodies, responses, links, security)
  -> emitted by an IClientEmitter, sequenced by ClientEmitDriver
  -> a list of GeneratedFile records, written to disk
```

Schema types are resolved behind `ISchemaTypeResolver`, so the emitter never hard-codes a language. The C# emitter uses the default resolver (a fully-qualified .NET type name), and the TypeScript engine supplies its own.

## Key types

- **`OpenApiWalkerBase`** is the abstract, version-neutral spec walk. It owns parameter merging, pointer building, security parsing, tag grouping, and server resolution. A per-version `OpenApiNNWalker` (in each version package) supplies only the strongly-typed model traversal.
- **`IClientEmitter`** is the emit contract: `PrepareContext`, `EmitRequestModule`, `EmitResponseModule`, `EmitClientInterface`, `EmitClientImplementation`, and the server-URI, links, and security surfaces. `OpenApiCSharpEmitterBase` is the C# implementation; `TypeScriptApiEmitter` (in the TypeScript package) is the TypeScript one.
- **`ClientEmitDriver`** sequences the walk and the emitter and collects the `GeneratedFile` results. It is language- and version-neutral.
- **`ISchemaTypeResolver`** and **`DefaultSchemaTypeResolver`** map a schema pointer to a language type name.
- **`OperationFilter`** provides glob include/exclude filtering over paths and channels. `OperationKind` distinguishes path, webhook, and callback operations.
- **`SchemaPointerBuilder`**, **`ParameterStyle`**, and **`ContentCategory`** are the shared primitives for schema references, the OpenAPI style/explode matrix, and body content categories.
- **`SecuritySchemes`**, **`SecurityRequirements`**, and **`EndpointSecurityConventions`** drive the emitted security-constant surface.

## Related packages

- `Corvus.Text.Json.OpenApi` is the run-time transport and serialization runtime the generated clients call.
- `Corvus.Text.Json.OpenApi30`, `.OpenApi31`, and `.OpenApi32` host the per-version walkers and C# emitters and provide the strongly-typed model types.
- `Corvus.Text.Json.OpenApi.TypeScript.CodeGeneration` is the TypeScript client emitter that plugs into this core.
- `Corvus.Text.Json` is the core library.
